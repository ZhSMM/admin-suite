# AI 对话增强 — 多级会话 + 导出 + 展示

## 现状（v0.6.9 baseline）

| 模块 | 文件 | 行为 |
|---|---|---|
| 多轮问答 | `src/views/ai/Chat.vue` | 单会话，消息仅存在 `ref`，刷新即丢 |
| 单轮工具 | `src/views/ai/{Explain,Summarize,Translate}.vue` | 输入框 → 输出框，无上下文 |
| 后端命令 | `src-tauri/src/commands/llm.rs::llm_chat/llm_chat_stream` | 接受 `messages` 参数，无状态 |
| 持久化 | `src-tauri/migrations/V10__seed_llm.sql` | `llm_providers/models/usage/budgets/defaults/fallback_config` — **没有任何会话/消息表** |
| i18n | `src/i18n/locales/*` | `ai.chat.*` 12 键，已有 |
| 鉴权 | `commands::auth::Session` | `state.fallback.llm_dir()` 模式 — 任何命令走 `require_perm` |

## 目标

1. **多级会话** — 一棵对话树（DAG），任意 assistant/用户节点可发散为多个分支，UI 可在兄弟节点间切换；切换会重写后续上下文（用选中的分支路径作 LLM 入参）。
2. **导出** — 单会话 → JSON（完整树）/ Markdown（活动路径扁平）/ HTML（可读视图）三种格式。
3. **展示** — 侧栏会话列表，主区树视图；活动分支高亮、隐藏分支灰显，兄弟节点有切换器。
4. **不破现有** — `ai.chat.*` 键不动；`ChatMessage` 接口不动；`Chat.vue` 默认仍可单聊，新功能通过侧栏开关启用；`Explain/Summarize/Translate` 不动。

## 方案

### 1. 数据库层（v0.7.0 迁移）

新文件：`src-tauri/migrations/V12__add_chat_history.sql`

```sql
CREATE TABLE IF NOT EXISTS chat_sessions (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    title           TEXT    NOT NULL DEFAULT '',          -- 首条 user 文本前 60 字符
    provider_id     TEXT    NOT NULL DEFAULT '',
    model_id        TEXT    NOT NULL DEFAULT '',
    created_at      TEXT    NOT NULL,
    updated_at      TEXT    NOT NULL,
    archived        INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
CREATE INDEX idx_chat_sessions_user_updated
    ON chat_sessions(user_id, updated_at DESC);

CREATE TABLE IF NOT EXISTS chat_messages (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id      INTEGER NOT NULL,
    parent_id       INTEGER,                              -- NULL ⇒ 根；同一 parent 多条 ⇒ 分支
    role            TEXT    NOT NULL,                     -- 'user' | 'assistant' | 'system'
    content         TEXT    NOT NULL,
    provider_id     TEXT    NOT NULL DEFAULT '',
    model_id        TEXT    NOT NULL DEFAULT '',
    status          TEXT    NOT NULL DEFAULT 'done',      -- 'done' | 'streaming' | 'error'
    error           TEXT,
    created_at      TEXT    NOT NULL,
    FOREIGN KEY (session_id) REFERENCES chat_sessions(id) ON DELETE CASCADE,
    FOREIGN KEY (parent_id)  REFERENCES chat_messages(id) ON DELETE CASCADE
);
CREATE INDEX idx_chat_messages_session
    ON chat_messages(session_id, id);
CREATE INDEX idx_chat_messages_parent
    ON chat_messages(parent_id);
```

> 备注：分支纯靠 `(session_id, parent_id)` 的同 parent 多行实现，不引入独立 branch 表。根节点 `parent_id IS NULL`。

### 2. Rust 命令

新文件：`src-tauri/src/commands/chat_history.rs`

| 命令 | 输入 | 输出 | 权限 |
|---|---|---|---|
| `chat_session_list` | `archived?` | `ChatSessionRow[]` | `llm:use` |
| `chat_session_create` | `{title?, providerId, modelId}` | `ChatSessionRow` | `llm:use` |
| `chat_session_update` | `{id, title?, archived?}` | `ChatSessionRow` | `llm:use` |
| `chat_session_delete` | `{id}` | `bool` | `llm:use` |
| `chat_session_get` | `{id}` | `{session, messages: ChatMessageNode[]}` —— 含 `children` 数组 | `llm:use` |
| `chat_message_append` | `{sessionId, parentId?, role, content, providerId, modelId, status?, error?}` | `ChatMessageNode` | `llm:use` |
| `chat_message_update` | `{id, content, status?, error?}` | `ChatMessageNode` | `llm:use` |
| `chat_message_delete` | `{id}` | `bool` | `llm:use` |
| `chat_export` | `{id, format: 'json'\|'markdown'\|'html', activePathIds: number[]?}` | `{filename, contentBase64}` | `llm:use` |

读取使用活动路径 `activePathIds`（根→叶），未提供时取「每个 parent 选最新子」形成的默认路径。

`sessionGet` 返回时构造 `ChatMessageNode { id, parent_id, role, content, ..., children: Vec<ChatMessageNode> }` —— 树形 JSON。

注册到 `src-tauri/src/lib.rs` 的 `tauri::generate_handler!` 列表。

### 4. UI（不替换 Chat.vue，外挂）

新文件：`src/views/ai/ChatHistory.vue` —— 直接把 Chat.vue 升级到树视图（增加侧栏），保留所有现有 props/state 兼容行为。

布局：

```
┌─────────────────────────────────────────────────────────────────┐
│ Page Header  [provider▼] [model▼]                  [save v0.7.0] │
├─────────────────┬───────────────────────────────────────────────┤
│ Sessions        │  [current path indicator: 5 of 7 msgs]        │
│ + New           │  ─────────────────────────────────            │
│ ─ Search ─      │  🧑 user: 我想用 LLM 找 bug              v0.7 │
│ ▸ 找 bug #3     │  🤖 assistant: ...  [↻ regenerate] [⎘ copy] │
│ ▸ Prompt 测试#1 │      ─────────                                  │
│ ▸ archived…     │      ↳ (branch · 2 siblings)  ◀  1 / 3  ▶    │
│                 │  🧑 user: 我用 llama-server 试过网络...        │
│                 │  🤖 assistant: ...                            │
├─────────────────┴───────────────────────────────────────────────┤
│  input textarea + [Send]                  [Clear] [Export ▼]   │
└─────────────────────────────────────────────────────────────────┘
```

**右侧消息操作**
- assistant 节点：[↻ regenerate] [⎘ copy] [✂ 隐藏分支以外]
- user 节点：[✎ edit & rebranch] [⎘ copy]
- 任意节点 hover：[⌫ delete sub-tree] [★ mark active branch from here]

**侧栏会话操作**
- 标题（inline 编辑） / 时间戳 / 消息计数 / provider-model
- 归档 / 恢复 / 删除 / 清空分支

### 8. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| 大表 `chat_messages` 爆炸 | `V12__add_chat_history.sql` 加 `idx_chat_messages_session`；UI 提供「归档」「删除」入口，删除是 `ON DELETE CASCADE` |
| 流式与持久化 race | v0.7 暂同步；流式后续版本先在 store 内 mark `status='streaming'` 再回调里 update |
| 切分支后，下游 message 在 UI 上「不存在」但 DB 中存在 | UI 用 `archived` 样式淡化显示；不物理删除，避免来回切换丢数据 |
| 旧 LLM 调用入参 | 直接用 `activePath` 重建的 `ChatMessage[]` 传给 `llmApi.chat`，零改动 |
| 旧 Chat.vue 用户习惯了单会话体验 | 把 `New Session` 默认隐藏，第一版进入是「当前会话」单聊；点 `History` 切换进侧栏视图 |
